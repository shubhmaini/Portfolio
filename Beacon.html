Beacon

Beacon – A commuter-friendly website designed to provide real-time updates on road conditions, including construction work, accidents, and closures. By selecting a route from a starting point to a destination, users receive relevant alerts to plan their journeys efficiently and enjoy a smoother commuting experience.

Goals / Purpose

1. Goal: To create a user-centric platform that delivers accurate, timely travel information.
2. Purpose: Help individuals and communities avoid delays by providing live updates and enabling better route planning.
3. Problem Solved: Reduces commuting stress, minimizes wasted time in traffic, and improves safety by alerting users about unexpected road events.

Tools & Technologies Used

1. Frontend: HTML5, CSS3, JavaScript, Bootstrap
2. Backend: Node.js, Express.js (if applicable)
3. Database: MongoDB / MySQL (if applicable)
4. Design: Figma for UI/UX mockups
5. APIs: Google Maps API, Traffic Data API (for real-time updates)

Process / Development Steps

1. Planning – Researched commuter needs and common traffic challenges. Defined core features like route selection, live alerts, and map integration.
2. Design – Created wireframes in Figma to outline page layouts. Designed an intuitive interface focusing on accessibility and readability.
3. Development – Built responsive frontend using HTML, CSS, and Bootstrap. Integrated Google Maps API to allow route selection. Connected traffic data API for real-time incident updates.
4. Testing – Tested across multiple devices for responsiveness. Checked API calls for accuracy and speed.
5. Deployment – Hosted website on a live server (e.g., Netlify/Heroku). Configured SSL for secure access.

Challenges & Solutions

1. Challenge: Ensuring secure account creation and transaction processing.
Solution: Implemented SSL encryption, input sanitization, and password hashing.
2. Challenge: Maintaining responsive design across devices.
Solution: Used Bootstrap’s grid system and media queries for adaptive layouts.
3. Challenge: Displaying detailed transaction history efficiently.
Solution: Used dynamic queries with pagination for fast loading.
